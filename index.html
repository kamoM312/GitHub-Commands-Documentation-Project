git init

Description

Turns any directory into a Git repository.

syntax

{ git init }

usage example

Creating a new local repository.

detailed explanation

To initialize a repository, Git creates a hidden directory called .git. That directory stores all of the objects and refs that Git uses and creates as a part of your project's history. This hidden .git directory is what separates a regular directory from a Git repository.

git pull

Description

Updates your current local working branch and all of the remote tracking branches.

syntax

{ git pull [repository url] }

usage example

Running the command on your branch before doing any work on it. To ensure that you have the most recent version in case other collaborators made changes on the branch.

detailed explanation

A combination of git fetch + git merge, updates some parts of your local repository with changes from the remote repository.

git add

Description

Adds new or changed files in your working directory to the Git staging area.

syntax

{ git add [file] }

usage example

After making changes to a file. In order to move it to the staging area, so it can be commited, use the git add command.

detailed explanation

Selects the specified file, and moves it to the staging area, marking it for inclusion in the next commit. You can select all files, a directory, specific files, or even specific parts of a file for staging and commit.

git commit

Description

Creates a commit, which is like a snapshot of your repository. These commits are snapshots of your entire repository at specific times.

syntax

{ git commit -m "[descriptive message]" }

usage example

After adding a file to the staging area, git commit can be run to specify the changes that were made to the file.

detailed explanation

Creates a commit, which is like a snapshot of your repository. These commits are snapshots of your entire repository at specific times. You should make new commits often, based around logical units of change. Over time, commits should tell a story of the history of your repository and how it came to be the way that it currently is. Commits include lots of metadata in addition to the contents and message, like the author, timestamp, and more.

git push

Description

Uploads all local branch commits to the corresponding remote branch.

syntax

{ git push }

usage example

After making your local commits, running git push will allow you to publish your changes to the remote repository.

detailed explanation

After you make and commit changes locally, you can share them with the remote repository using git push. Pushing changes to the remote makes your commits accessible to others who you may be collaborating with.

git remote

Description

Manages the set of remotes that you are tracking with your local repository.

syntax

{ git remote -v/ git remote add [name] [URL]/ git remote remove [name] }

usage example

Listing the current remotes associated with your local repository.

detailed explanation

Manages the set of remotes that you are tracking with your local repository. Allows you to view current repository remotes, add remotes and delete remotes.

git checkout

Description

Switches from the current branch to the specified branch.

syntax

{ git checkout [branch-name] }

usage example

You are currently working on a development branch with collaborators. You need to test some features, you can switch to a testing branch. In order to perform tests without affecting the other collaborators.

detailed explanation

Allows you to switch branches by specifying the name of the branch that you want to switch to.
